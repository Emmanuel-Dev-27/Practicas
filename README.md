# 📚 Dev-Labs: Repositorio Central de Aprendizaje

**El laboratorio de experimentación, algoritmos y prototipado.**

Este repositorio es una colección organizada de mis prácticas, retos y experimentos en diversos lenguajes y motores de desarrollo. Aquí documento mi evolución técnica, desde fundamentos de programación hasta arquitecturas más complejas.

---

## 🧭 Mapa del Laboratorio

He organizado mis prácticas por ecosistemas para facilitar la navegación:

### [🐍 Python (Backend & Scripting)](./Python)
Colección de scripts enfocados en lógica, automatización y manejo de datos. Incluye APIs con Flask, FastAPI y persistencia en MySQL/MongoDB.
> **Conceptos:** Manejo de excepciones, entornos virtuales, decoradores, JWT y estructuras de datos.

### [🎮 Godot Engine (Game Dev)](./godot)
Implementación de mecánicas core y prototipos de juegos funcionales.
> **Conceptos:** Signals, Autoloads, GDScript avanzado, sistemas de movimiento y físicas 2D.

### [🟦 C / Low Level](./c)
Ejercicios de bajo nivel enfocados en la eficiencia y el control total del hardware.
> **Conceptos:** Gestión de memoria dinámica, punteros, algoritmos matriciales y encriptación XOR.

### [🌐 JavaScript & Frontend](./JavaScript)
Prototipos interactivos y aplicaciones web modernas utilizando Vanilla JS y el ecosistema de React.
> **Conceptos:** Hooks, manejo del DOM, Canvas API, y servicios Full Stack (React + Flask).

---

## 🛠️ Herramientas y Metodología
Para mantener este laboratorio eficiente, utilizo:

*   **Estructura de Monorepo:** Centralización para facilitar la búsqueda de soluciones reutilizables.
*   **Documentación Detallada:** Cada subproyecto cuenta con su propio `README.md` detallando los conceptos aplicados.
*   **Gestión Unificada:** Historial centralizado para dar visibilidad a la progresión en múltiples tecnologías.

## 📈 Estadísticas de Aprendizaje
*   **Lenguajes dominantes:** C, Python, GDScript, JavaScript.
*   **Áreas de enfoque:** Lógica de programación, Game Design, Estructuras de Datos, Arquitecturas de API.
